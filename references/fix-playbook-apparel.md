# 服装类目 Listing 修复剧本 (Fix Playbook)

这一篇是给运营美工执行用的,Claude 在产出修改建议时**必须**按这个剧本的格式给具体动作,不允许写"提升内容质量"这种话。

每条建议必须有:**改哪里 + 改成什么 + 为什么改 + A10/COSMO/Rufus 各自获益**。

---

## 主图 7 张完整剧本

### 图 1(主图)—— 转化第一关

**目标**:让买家在搜索结果页 0.5 秒内看清版型轮廓 + 颜色

**必备元素**:

- 白底(平台规则)
- 模特/平铺/挂拍三选一,服装品类强烈推荐**模特**(转化高 30% 以上)
- 正面或 45 度角
- 完整露出版型轮廓(连衣裙必须看到下摆,T 恤必须看到肩线和长度)
- **绝对不允许**文字、Logo、水印、对比图(否则被卡审)

**美工常见错误**:

- ❌ 主图用花里胡哨场景,版型反而看不清
- ❌ 主图用平铺,转化掉 30%
- ❌ 主图打文字标 "Best Seller"

### 图 2 —— 上身正面

**目标**:让买家想象"我穿是什么样"

**必备元素**:

- 模特正面全身,版型自然垂坠
- 背景可以淡化的场景背景(室内、自然光)
- 无文字
- 配饰极简(不抢戏)

### 图 3 —— 上身背面 + 侧面

**目标**:回答"后面/侧面长什么样"——这是 Rufus 频繁被问的角度

**必备元素**:

- 同一模特,背面 + 侧面拼图,或两图分开
- 露出口袋位置(如果有)、拉链位置、剪裁特点(如收腰、A 字摆)

### 图 4 —— 面料近景 + 文字标注 ★★★★★

**这一张是 Rufus 最常引用的图**。

**必备元素**:

- 面料微距特写(露出织纹)
- 文字标注**至少包括**:
  - 成分(95% Cotton, 5% Spandex)
  - 克重(如有,200 GSM)
  - 弹力(4-Way Stretch / 30% Stretch)
  - 是否衬里(Fully Lined / Not Lined)
  - 是否透(Opaque / Semi-sheer)
- 可选:面料抗皱演示(揉一下展开)

**为什么**:Rufus 答 "What's it made of"、"Is it stretchy"、"Is it see-through"、"Will it shrink" 这些问题时,会扫文字标注。Listing 没图 4,这些问题 Rufus 只能从 review 里找,质量不可控。

### 图 5 —— 尺码表 + 模特身高对照 ★★★★★

**这是退货率第一杀手**。

**必备元素**:

- 完整尺码表(英寸 + 厘米双单位)
- 模特身高(如 5'8" / 175cm)
- 模特穿着尺码(如 Wearing Size S)
- 模特三围或胸/腰/臀实测(可选但强推)
- "How to Measure" 小图示(指明胸围/腰围怎么量)
- 如果尺码偏大/偏小,直接标 "Runs True to Size" 或 "We Recommend Sizing Up"

**美工常见错误**:

- ❌ 尺码表用美式 0/2/4/6 不带英寸/cm 数字 — 海外华人/欧洲买家完全看不懂
- ❌ 不写模特身高
- ❌ 尺码表只标 chest/waist/hip 不标长度

### 图 6 —— 场景图(3-4 个场景拼图) ★★★★ COSMO 友好

**目标**:用图片让 COSMO 理解这件衣服的使用场景

**必备元素**:

- 3-4 个不同场景小图拼成一张,每图带场景标签:
  - "Office Meeting"
  - "Brunch with Friends"
  - "Weekend Errands"
  - "Date Night"
- 每个场景模特表情/动作不同(避免感觉是一张图复制)

**为什么**:COSMO 会把这件衣服跟 "office dress"、"brunch dress" 等长尾意图关联起来。

### 图 7 —— FAQ 图 ★★★★ Rufus 友好

**这一张是 Rufus 友好型新增图**,不是传统美工标配,但回报极高。

**必备元素**:

- "Frequently Asked" 标题
- 5-6 条 Q&A 文本对,每条 ≤ 30 字:
  - "Will it shrink? — No, pre-shrunk fabric."
  - "Machine washable? — Yes, cold water inside-out."
  - "Run true to size? — Yes, but for between sizes, size up."
  - "Is it see-through? — No, fully lined."
  - "Stretchy? — 4-way stretch, comfortable for long wear."
  - "Iron needed? — Wrinkle-resistant, light steam if needed."

**为什么**:Rufus 把图文都当文本扫描,FAQ 图被引用率极高。一张图能解决 5-6 个高频疑问。

## A+ 内容 7 模块结构

### Module 1 — Brand Story

- 1 段品牌故事(50-80 字)
- 图:品牌创始场景 / 工坊 / 设计师
- 文字必须包含**为谁设计**(working moms / busy professionals / petite women)
- COSMO 高度依赖这个模块识别人群

### Module 2 — Real Models, Real Bodies

- 3-4 个不同身材模特同款实穿(petite / regular / tall / plus 各一)
- 每张图标:身高 / 穿着尺码 / 实际三围
- 这是 Rufus 答"5'2 穿这件好不好看"的核心证据

### Module 3 — Fabric & Craftsmanship

- 面料近景 + 工艺细节(走线、面料织法、染色)
- 数据图卡:成分%、克重、弹力%、是否预缩
- Rufus 答面料问题主要从这里引

### Module 4 — Size Guide

- 详细尺码表(双单位)
- "How to Measure" 步骤图
- "Between Sizes? — We recommend..." 建议
- 不同身高如何选码的指引

### Module 5 — Compare Our Styles(自家系列对比)

- 2-3 列横向对比表:本款 vs 自家其他款
- 对比维度:版型 / 面料 / 长度 / 场景 / 价位
- **不要点名竞品**,容易被申诉
- Rufus 答"系列内哪款更适合我"时直接引用这个表

### Module 6 — FAQ

- 5-8 条问答,覆盖洗护、尺码、场合、面料
- **每条答案必须有具体事实**,不要"我们尽力做到最好"

### Module 7 — Care & Origin

- 洗护说明图标(机洗冷水 / 不漂白 / 低温烘 / 不熨烫等)
- 产地 / 设计国
- 退换政策简述

## Bullet 5 条服装专用模板

### Bullet 1 — 版型 + 人群定位 + 核心场景

**模板**:`[FIT TYPE] [GARMENT] DESIGNED FOR [PERSONA] — [Lead with silhouette and the problem it solves for a specific buyer in 1-2 sentences. Mention the primary occasion.]`

**示例**:`RELAXED-FIT MIDI DRESS FOR WORKING MOMS — A flowy A-line silhouette that hides postpartum tummy without sacrificing style. Designed to take you from school drop-off to dinner without changing.`

为什么:COSMO 需要场景+人群,Rufus 需要具体问题陈述。

### Bullet 2 — 面料数据

**模板**:`PREMIUM [FABRIC %], [WEIGHT/STRETCH/LINING] — [Describe how the fabric feels, breathability, and care advantage.]`

**示例**:`SOFT 95% MODAL + 5% SPANDEX, 220 GSM, 4-WAY STRETCH — Buttery feel, breathable for warm weather, holds shape after multiple washes. Fully lined for opacity.`

### Bullet 3 — 尺码建议 + 实测对照

**模板**:`TRUE-TO-SIZE FIT — Model is [HEIGHT] wearing [SIZE]. [If runs small/large, say so.] [Size advice for between-sizes.] See size chart in image [N].`

**示例**:`TRUE TO SIZE — Model is 5'8" / 173cm wearing Size S. If you're between sizes, size down for a closer fit, up for relaxed. Detailed measurements in image 5.`

### Bullet 4 — 场景列举(3-5 个)

**模板**:`VERSATILE FOR [OCCASION 1], [OCCASION 2], [OCCASION 3] — [Brief styling note linking to image.]`

**示例**:`PERFECT FOR OFFICE, BRUNCH, DATE NIGHTS, AND WEDDING GUESTS — Pair with sneakers for casual, heels for evening. See styled looks in image 6.`

为什么:COSMO 命中长尾意图("dress for X"),Rufus 引用具体场景。

### Bullet 5 — 洗护 + 退换

**模板**:`EASY CARE — Machine wash cold inside-out, tumble dry low, no ironing needed. [Wrinkle/shrink claim.] 30-day hassle-free returns if size doesn't fit.`

**示例**:`MACHINE WASHABLE — Cold wash inside-out, tumble dry low. Wrinkle-resistant for travel. Pre-shrunk fabric — no shrinkage after wash. 30-day free returns.`

## Title 服装类目模板

### 保守版(改动小,不冒险)

`[BRAND] [GENDER] [KEY FIT/STYLE 1] [GARMENT] [KEY FEATURE 1] [KEY FEATURE 2]`

例:`AAA Women's Wrap Midi Dress with Pockets, Stretch Knit`

### 平衡版(推荐 — A10 + COSMO 都顾)

`[BRAND] [GENDER] [PERSONA-OR-FIT] [GARMENT] [KEY FEATURE], [PRIMARY OCCASION]`

例:`AAA Women's Petite Wrap Midi Dress with Pockets, Office to Brunch`

### 激进版(COSMO 优先,A10 略让)

`[BRAND] [PERSONA] [GARMENT], [FIT/STYLE], for [OCCASION 1] and [OCCASION 2]`

例:`AAA Petite Women's Wrap Dress, Stretch Midi for Office and Brunch`

**Title 改动建议**:除非现在 title 完全是关键词堆砌,否则**优先用平衡版**。Title 大改有 search rank 重置风险,要稳。

## 后台 Search Terms(250 字符)

按这个分配:

- 30%:核心品类关键词(同义/复数/拼写变体)
- 30%:场景词(occasion 类长尾)
- 20%:人群词(petite / tall / plus / curvy / pregnant 等)
- 20%:特征词(stretchy / pockets / midi / wrap 等)

不要重复 title 已有的词。

## Q&A 区(老客户/亲友引导)

### 鼓励老客户主动问的问题(然后店家回答):

- "I'm 5'3 and 130 lbs, what size?"
- "Does this hold up after multiple washes?"
- "Can I wear this to a beach wedding?"

### 店家回答模板:

- 必须包含**具体数据**(身高、尺码、洗后效果)
- 链接到 Listing 图位号("see image 5 for size chart")
- 不要写"please contact us",直接答

---

## Review 引导(合规框架内)

### 售后卡 / 邮件可以写

- ✅ "我们很想知道你穿这件衣服去了什么场合,可以分享一下吗?"
- ✅ "如果方便的话,可以提一下你的身高和你买的尺码,这能帮其他买家做选择"
- ✅ "你最喜欢这件衣服的哪个细节?"

### 不可以写

- ❌ 任何暗示评分(五星/好评)的内容
- ❌ 提供任何返现/优惠换 review
- ❌ 让买家在留 review 前先联系你

## 修改输出格式(给运营美工的标准格式)

每条修改建议必须按这个模板写,Claude 产出报告时严格遵守:

```
【修改 N】优先级:5 critical
位置:主图第 5 张
现状:只有尺码表,没有模特身高对照
改成:在尺码表右侧加"Model: 5'8" / 173cm wearing S, Bust 33", Waist 25", Hip 35""
为什么:Rufus 反复被问"5'2/5'5/5'8 适合穿什么尺码",目前 review 里有人吐槽"模特多高都没说"
算法对齐:A10 中(降退货 → 长期 BSR)| COSMO 低 | Rufus 高(直接被引)
执行人:美工
预计工时:30 分钟
```

如果有多条修改集中在同一位置(如主图 5 张要改),按位置归并,但每条改动单独编号。
